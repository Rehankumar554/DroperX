# Security Policy

At DroperX, the privacy and security of our users are our highest priorities. Since the application inherently deals with peer-to-peer file transfers, we take vulnerabilities extremely seriously.

## Supported Versions

Currently, the main branch of DroperX is the only supported version for security updates.

| Version | Supported          |
| ------- | ------------------ |
| `main`  | :white_check_mark: |
| Older   | :x:                |

## Reporting a Vulnerability

If you discover a security vulnerability within DroperX, please **DO NOT** create a public GitHub Issue. This prevents malicious actors from exploiting the vulnerability before we can issue a patch.

Instead, please send an email directly to our security team at:
**[rksdevelopmentstudio@gmail.com](mailto:rksdevelopmentstudio@gmail.com)**

### What to include in your report:
- A description of the vulnerability.
- Clear steps to reproduce the issue.
- Any relevant logs, screenshots, or code snippets.

### Our Response:
- We will acknowledge receipt of your vulnerability report within 48 hours.
- We will work with you to understand and resolve the issue quickly.
- We may ask you to hold off on publishing details about the vulnerability until we have pushed a fix to the main branch.

Thank you for helping us keep DroperX secure for everyone!
