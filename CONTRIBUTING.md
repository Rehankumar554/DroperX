# Contributing to DroperX

First off, thank you for considering contributing to **DroperX**! It's people like you that make the open-source community such a great place to learn, inspire, and create.

## 🤝 How Can I Contribute?

### 1. Reporting Bugs
This section guides you through submitting a bug report for DroperX.
- **Ensure the bug was not already reported** by searching on GitHub under [Issues](https://github.com/Rehankumar554/DroperX/issues).
- If you're unable to find an open issue addressing the problem, open a new one.
- Describe the bug precisely, including OS, browser version, and steps to reproduce.

### 2. Suggesting Enhancements
Enhancement suggestions are highly welcome.
- Open an Issue with the label `enhancement`.
- Clearly describe the feature you want to add.
- Explain why this enhancement would be useful to most DroperX users.

### 3. Pull Requests (Code Contributions)
If you want to contribute directly to the codebase:

1. **Fork the repository** on GitHub.
2. **Clone your fork** locally: `git clone https://github.com/YourUsername/DroperX.git`
3. **Create a new branch** for your feature or bugfix: `git checkout -b feature/your-feature-name`
4. **Make your changes** following the project's current CSS structure and Vanilla JS setup.
5. **Test your changes** locally on your web server (e.g., XAMPP) to ensure no regressions occur.
6. **Commit your changes** with clear, descriptive commit messages: `git commit -m "Add new feature X"`
7. **Push to your fork**: `git push origin feature/your-feature-name`
8. **Open a Pull Request (PR)** against the `main` branch of the original DroperX repository.

## 🎨 Code Guidelines

- **HTML/CSS:** We use pure HTML and custom CSS. Do not use external UI frameworks like Tailwind or Bootstrap unless previously discussed in an issue.
- **JavaScript:** Vanilla JS. Ensure your code does not break the asynchronous nature of the WebRTC `PeerJS` connections.
- **Design:** Ensure your UI additions adhere to the glassmorphic, modern, and dark-mode aesthetic of the app.

We look forward to reviewing your Pull Request!
